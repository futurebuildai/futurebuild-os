# Phase 18: ERP Transition Roadmap
**Master Sprint Plan**

This roadmap defines the actionable development sprints to achieve the Autonomous Enterprise Ecosystem. It adheres to L7 Zero-Trust limits, maintaining strict boundaries between probabilistic AI (FB-Brain) and deterministic physics (FutureBuild OS).

## Epic 1: Database Spine Expansion (FutureBuild OS)
- **Sprint 1.1:** Extend PostgreSQL models for Corporate Financials, supporting cross-project budget rollups.
- **Sprint 1.2:** Implement HR and Employee Management DB schemas (prevailing wage tracking, certifications).
- **Sprint 1.3:** Develop Equipment & Fleet Asset Management (EAM) tracking schemas and allocation logic.

## Epic 2: UI/UX Overhaul (FutureBuild OS)
- **Sprint 2.1:** Integrate the GableLBM Design System globally across the web app.
- **Sprint 2.2:** Build the Voice-First Field Portal (mobile-native views, giant touch targets, voice-to-text offline queue).
- **Sprint 2.3:** Implement `fb-settings-brain.ts` to bridge the OS and FB-Brain, displaying active integrations and agent logs natively.

## Epic 3: FB-Brain Agent Studio Upgrade (FB-Brain)
- **Sprint 3.1:** Evolve the `@xyflow/react` canvas into the drag-and-drop Agent Studio.
- **Sprint 3.2:** Implement Maestro Co-Pilot intent parser for workflow generation.
- **Sprint 3.3:** Build MCP client connections for GableERP and QuickBooks.

## Epic 4: OS-to-Brain Bridge (A2A & MCP)
- **Sprint 4.1:** Establish secure bidirectional webhooks between OS and Brain.
- **Sprint 4.2:** Implement the centralized A2A logging and "Why-Trails" visualization API.
- **Sprint 4.3:** L7 Zero-Trust security audit of cross-platform execution borders.
